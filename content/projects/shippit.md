## Context
Shippit is an AI-native software consultancy that embeds with client teams to ship production software fast: AI-native development, blockchain engineering, full-stack implementation. I joined in late 2025 as Design Engineer, working front-end and design systems across client engagements and Shippit's own products.

## What I did
Three threads:

- Design engineering on Shippit's client engagements (BWB Tokenization, Founderhaus Base, Learn to Fly).
- The current shippit.app website end-to-end.
- Experimental Claude plugins for the team's AI tooling.

The through-line: clean front-end, design systems that scale, AI rules that govern how agents propose code.

## Outcome
Active engagement. Client projects shipped or in flight. shippit.app live. Claude plugins still experimental, exploring how far we can push design and agent workflows inside the IDE.

## Deep Dive

### Client Projects

- **BWB Tokenization:** Front-end lead on a real-estate tokenization platform. Built with React Router 7, Convex, TypeScript, Tailwind, Shadcn, Ethers, Privy. Defined 13 milestones (MS-01–MS-13), authored a 52-document system, defined 7 personas, designed the authorization architecture, built an AI-optimized INDEX.md.
- **Founderhaus Base:** Live Farcaster Mini App on Base. Full front-end build.
- **Learn to Fly:** Expo / React Native refactor with NativeWind, RN Primitives, design system, and component refactor.

### shippit.app Website

Built the current production website end-to-end: brand alignment, product positioning, design system applied front-to-back.

### Claude Plugins (experimental)

Building experimental Claude plugins for the team. Exploring design system enforcement, agent rules that guide how Claude/Cursor agents propose code, and internal workflows that keep design and code aligned inside the IDE. Still early; treating it as R&D for what AI tooling can do for design engineering teams.
